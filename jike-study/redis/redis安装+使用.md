###  一、安装redis

1、官网下载压缩包https://redis.io/download

or brew install redis(太慢了...)

2、解压

```
tar zxvf redis-6.0.9.tar.gz
```

3、复制到usr/local目录下

```
mv redis-6.0.9 /usr/local/
```

若此时报权限不足可使用:sudo mv redis-6.0.9 /usr/local/

4、切换到对应目录,编译测试

```
cd /usr/local/redis-6.0.9
```

sudo make test[我这里出现了一个errno,但是没管它]

编译安装

```
sudo make install
```

5、启动

```
redis-server
```

6、查看redis服务是否启动

```
ps aux|grep redis
```

7、redis-cli连接redis服务

```
redis-cli -h 127.0.0.1 -p 6379
```

### 二、redis使用

语法:redis 127.0.0.1:6379> COMMAND KEY_NAME

```
127.0.0.1:6379> set name aogo
OK
127.0.0.1:6379> get name
"aogo"
127.0.0.1:6379>
```

#### 2、hash类型(hmset/hmget)

Redis hash 是一个 string 类型的 field（字段） 和 value（值） 的映射表，hash 特别适合用于存储对象。

```
127.0.0.1:6379> hmset user name aogo age 10 gender girl hobby baseketball
OK
127.0.0.1:6379> hgetall user
1) "name"
2) "aogo"
3) "age"
4) "10"
5) "gender"
6) "girl"
7) "hobby"
8) "baseketball"
127.0.0.1:6379> hmget user name
1) "aogo"
127.0.0.1:6379> hmget user age
1) "10"
```

#### 3、list类型(lpush/lrange)

Redis列表是简单的字符串列表，按照插入顺序排序。你可以添加一个元素到列表的头部（左边）或者尾部（右边）

```
27.0.0.1:6379> lpush middwears redis
(integer) 1
127.0.0.1:6379> lpush middwears rabbitmq
(integer) 2
127.0.0.1:6379> lpush middwears kaffa
(integer) 3
127.0.0.1:6379> lrange middwears 0 2
1) "kaffa"
2) "rabbitmq"
3) "redis"
127.0.0.1:6379> lrange middwears 0 1
1) "kaffa"
2) "rabbitmq"
```

#### 4、set类型(sadd/smembers)

Redis 的 Set 是 String 类型的无序集合。集合成员是唯一的，这就意味着集合中不能出现重复的数据。

Redis 中集合是通过哈希表实现的，所以添加，删除，查找的复杂度都是 O(1)。

```
127.0.0.1:6379> sadd weeks Monday
(integer) 1
127.0.0.1:6379> sadd weeks Tuesday
(integer) 1
127.0.0.1:6379> sadd weeks Wednesday
(integer) 1
127.0.0.1:6379> sadd weeks Monday
(integer) 0
127.0.0.1:6379> smembers weeks
1) "Tuesday"
2) "Monday"
3) "Wednesday"
```

#### 5、sorted set类型(zadd/zrange)

Redis 有序集合和集合一样也是 string 类型元素的集合,且不允许重复的成员。

区别是sorted set每个元素都会关联一个 double 类型的分数。redis 正是通过分数来为集合中的成员进行从小到大的排序。

有序集合的成员是唯一的,但分数(score)却可以重复。

集合是通过哈希表实现的，所以添加，删除，查找的复杂度都是 O(1)

```
127.0.0.1:6379> zadd students 1 aogo
(integer) 1
127.0.0.1:6379> zadd students 3 Clore
(integer) 1
127.0.0.1:6379> zadd students 2 Bob
(integer) 1
127.0.0.1:6379> zadd students 2 Zoe
(integer) 1
127.0.0.1:6379> zrange students 0 10 withscores
1) "aogo"
2) "1"
3) "Bob"
4) "2"
5) "Zoe"
6) "2"
7) "Clore"
8) "3"
```

#### 6、HyperLogLog(pfadd/pfcount)

Redis HyperLogLog 是用来做基数统计的算法，优点是在输入元素的数量或者体积非常非常大时，计算基数所需的空间总是固定的、并且是很小的。

在 Redis 里面，每个 HyperLogLog 键只需要花费 12 KB 内存，就可以计算接近 2^64 个不同元素的基 数。这和计算基数时，元素越多耗费内存就越多的集合形成鲜明对比。

 HyperLogLog 只会根据输入元素来计算基数，而不会储存输入元素本身，所以 HyperLogLog 不能像集合那样，返回输入的各个元素。

```
127.0.0.1:6379> pfadd skills java
(integer) 1
127.0.0.1:6379> pfadd skills python
(integer) 1
127.0.0.1:6379> pfadd skills go
(integer) 1
127.0.0.1:6379> pfadd skills redis
(integer) 1
127.0.0.1:6379> pfcount skills
(integer) 4
```

#### 7、订阅/消费

Redis 发布订阅 (pub/sub) 是一种消息通信模式：发送者 (pub) 发送消息，订阅者 (sub) 接收消息。

Redis 客户端可以订阅任意数量的频道。

下图展示了频道 channel1 ， 以及订阅这个频道的三个客户端 —— client2 、 client5 和 client1 之间的关系：

![img](https://www.runoob.com/wp-content/uploads/2014/11/pubsub1.png)

当有新消息通过 PUBLISH 命令发送给频道 channel1 时， 这个消息就会被发送给订阅它的三个客户端：

![img](https://www.runoob.com/wp-content/uploads/2014/11/pubsub2.png)


#### 8、redis事务

redis 事务可以一次执行多个命令， 并且带有以下三个重要的保证：

- 批量操作在发送 EXEC 命令前被放入队列缓存。
- 收到 EXEC 命令后进入事务执行，事务中任意命令执行失败，其余的命令依然被执行。
- 在事务执行过程，其他客户端提交的命令请求不会插入到事务执行命令序列中。

一个事务从开始到执行会经历以下三个阶段：

- 开始事务。
- 命令入队。
- 执行事务。



以multi开启事务,将多个指令存储到队列中,以exec指令触发事务,一并执行事务中的所有命令.



```
127.0.0.1:6379> multi
OK
127.0.0.1:6379> set book "master go in 21 days"
QUEUED
127.0.0.1:6379> get book
QUEUED
127.0.0.1:6379> sadd tag "go" "programming" "it"
QUEUED
127.0.0.1:6379> smembers tag
QUEUED
127.0.0.1:6379> exec
1) OK
2) "master go in 21 days"
3) (integer) 3
4) 1) "it"
   2) "programming"
   3) "go"
127.0.0.1:6379>
```

单个 Redis 命令的执行是原子性的，但 Redis 没有在事务上增加任何维持原子性的机制，所以 Redis 事务的执行并不是原子性的。

事务可以理解为一个打包的批量执行脚本，但批量指令并非原子化的操作，中间某条指令的失败不会导致前面已做指令的回滚，也不会造成后续的指令不做。



##### Redis 事务命令

下表列出了 redis 事务的相关命令：

| 序号 | 命令及描述                                                   |
| :--- | :----------------------------------------------------------- |
| 1    | [DISCARD](https://www.runoob.com/redis/transactions-discard.html) 取消事务，放弃执行事务块内的所有命令。 |
| 2    | [EXEC](https://www.runoob.com/redis/transactions-exec.html) 执行所有事务块内的命令。 |
| 3    | [MULTI](https://www.runoob.com/redis/transactions-multi.html) 标记一个事务块的开始。 |
| 4    | [UNWATCH](https://www.runoob.com/redis/transactions-unwatch.html) 取消 WATCH 命令对所有 key 的监视。 |
| 5    | [WATCH key [key ...\]](https://www.runoob.com/redis/transactions-watch.html) 监视一个(或多个) key ，如果在事务执行之前这个(或这些) key 被其他命令所改动，那么事务将被打断。 |

#### 9、Redis 安全

```
127.0.0.1:6379> config get requirepass
1) "requirepass"
2) ""
127.0.0.1:6379> config set requirepass aogo
OK
127.0.0.1:6379> config get requirepass
1) "requirepass"
2) "aogo"
127.0.0.1:6379> auth aogo
OK
```

**AUTH** 命令基本语法格式如下：

```
127.0.0.1:6379> AUTH password
```
